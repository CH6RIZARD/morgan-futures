# morgan futures

Futures side of the same headache. Rhythmic API and data layer meant to talk to the futures tape the way the crypto bot talks to exchange charts.

There is a UI living in a local folder called ezer morgan futures that is supposed to be the real face of this. Right now this GitHub repo is basically doing backend duty for that UI and the wiring honest to god is not clean so do not read this as a glossy shipped product. It is the data spine while I untangle what should live where.

Portfolio wise there is funded futures work (LucidFlex, MGC, MYM, consistency discipline) floating behind the same worldview as the rest of my trading stack. Scenario reads, liquidity weather, not astrology.

**If you are cloning expecting a polished app you will be disappointed.** If you are ok with this is the messy middle while architecture catches up you are in the right place.
